The 8086 generates a 1 MB address by combining a 16-bit segment register value and a 16-bit offset value to produce a 20-bit physical address. This is done through the formula: Physical Address = (Segment Register × 16) + Offset. This segmented approach allows the 8086 to have a large address space while only needing 16-bit registers to hold the segment and offset values.          How the address is generated     20-bit address bus: The 8086 has a 20-bit address bus, allowing it to access \(2^{20}\) bytes, which equals 1,048,576 bytes or 1 MB of memory.  Memory segmentation: Memory is divided into 64 KB segments.  Segment registers: There are four 16-bit segment registers (Code Segment (CS), Data Segment (DS), Stack Segment (SS), and Extra Segment (ES)) that hold the starting address of a memory segment.  Offset: A 16-bit offset (also called the effective address) is used to point to a specific location within that segment.  Address calculation: To get the final 20-bit physical address, the 16-bit segment register value is shifted left by four bits (equivalent to multiplying by 16) and then the 16-bit offset value is added to it.  4 Intel 8086 | PDF
